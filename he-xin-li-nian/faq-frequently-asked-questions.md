# 常见问题解答（FAQ）

#### 1. Tizi是一个分叉吗？

不；Tizi是从零开始构建的。然而，它确实扩展了其他DeFi建设者的工作。TD的多链重置机制受到Tangible DAO的启发。而Tizi的金库和farm合约设计，虽然不是分叉，但是理念受到了像Yearn这样的收益聚合器项目的启发。

#### 2. Tizi采用了哪种跨链桥接技术？

桥在Tizi中有两个主要用途：一是传递信息（例如，用于重置计算），二是转移基础资产。Tizi并不依赖于特定的桥技术，以下是它目前所使用的桥：

| ----------- | -------------------- | -------------------- |
| Circle CCTP | :heavy\_check\_mark: |                      |
| Axelar      |                      | :heavy\_check\_mark: |
| Layer Zero  | :heavy\_check\_mark: | :heavy\_check\_mark: |

#### 3. Wen token?

我们没有治理代币，也没有发行治理代币的计划。任何提供出售或赠送这类代币的人都是骗子。

#### 4. 风险因素以及解决方案是什么？

*   跨链桥偶尔会停止正常运作。在这种情况下，资金将在当前链上暂时锁定，直到桥的状态恢复正常，资金才可跨链转移。

    Tizi的解决方案是：如果主链 Base 上的资金池还有资金储备，申请人可立即提取资金。当主链资金池已被全部赎回，而其他资金在其他链上锁定时，Tizi 在等待期间为受影响用户提供 NFT 凭证。这些 NFT 也可通过市场交易提前变现。
*   如果USDC在市场上大幅贬值，那么由于TD是由USDC和与USDC保持Delta中性的其他投资支持的，我们可以预期TD的市场价值也会随之下降。如果这种脱钩是暂时的，同样可以预期TD会随着USDC的价值恢复而恢复。

    解决方案：不需要

#### 5. Tizi如何规避中心化风险？

首先我们会确保NFT中待取USDC的优先提取，在NFT中有待取USDC的情况下，管理员不可以从mainVault中转移超过该数量的USDC，并且不可以部署新策略或新vault，管理员可以通过LayerZero在各链上传递流动性信息。部署新策略后，不会立即使用，需要至少等待3天并且确保流动性充足，在此期间用户可以对策略进行审计，如果用户对新的策略持异议，便能够在它生效前选择退出，并且除非已退出的用户获得相应补偿，否则策略将不会使用。升级合约的权限被赋予TimeLock合约，升级不会立刻进行，同样至少需要等待三天并且确保流动性充足，并且确保没有其他地址拥有升级权限。在未来我们会将管理员权限转移至多签钱包，进一步减少中心化风险。
